v2 · April 2026 The web is broken. We removed the broken parts. A design system with zero dependencies, zero build step, zero npm. Just web standards — the way the browser intended. Start building Read the manifesto ↓ 8 kB gzipped JS. 19 components. 0 dependencies. Measured, not estimated. The problem A button should not require a build pipeline. The typical SaaS page ships 2 MB of JavaScript before it paints. 38 npm packages to render a rounded rectangle. 14 config files to style it. Slower than the same page built in 2008. The web got worse. Magrathea isn't better — it's what's left when you remove what got worse. First principles Seven rules. No exceptions.
01 Physics before code. The browser renders pixels. Everything above is a layer that can rot.
02 No npm. Lockfiles lie. Supply chains die. Every dependency is a promise the author will still care in three years.
03 HTML is the interface. Attributes beat classes. Components beat stylesheets. What you read is what you write.
04 CSS is how pixels get color. Not a framework. Not a methodology. Not a JavaScript problem.
05 JavaScript is a tool, not a religion. Reach for it last. The browser has native popovers, dialogs, form validation, routing, and view transitions now.
06 Web standards outlive companies. Your framework vendor might pivot. The W3C won't. Bet on the browser — it's the one thing that can't be deprecated.
07 Ship what you mean. Every kilobyte costs someone — battery, bandwidth, attention, carbon. Ship less. Mean more.
Footnote In the spirit of Dieter Rams' Less, but better and the Swedish principle of lagom: not sparse, just enough.

Code an agent writes today still runs in 2035 — no migration guide required. Copy, paste, works No "now run npm install." The page loads. The agent's next step is a diff, not a build.
